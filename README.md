# rutherford_project

Первый проект созданный с помощью OpenGL (glfw, glad, glm).
В целом проект

Описание проекта:
Этот проект представляет собой мини-движок на основе OpenGL 3.3 Core Profile, реализующий простую 2D симуляцию частиц с использованием матриц трансформации и шейдеров. Он демонстрирует работу с вершинными буферами, VAO/VBO, uniform-переменными и базовую физику движения частиц.

Основные возможности:
Рендеринг объектов через VBO и VAO.
Отрисовка ядра и множества частиц с индивидуальными позициями, скоростями и масштабом.
Использование GLM для работы с матрицами трансформации (model и projection).
Динамическое обновление частиц с учётом времени (deltaTime).
Поддержка шейдеров для определения цвета и трансформаций объектов.
2D проекция с корректным соотношением сторон (orthographic projection).

Структура проекта:
src/main.cpp - основной цикл рендеринга и логика спавна частиц.
src/shaders.cpp/h - загрузка и использование шейдеров.
src/physics.h - управление движением частиц и физика их взаимодействия с ядром.
src/vertices.h / src/geometry.h - генерация вершинных данных для объектов (круги).
shaders/ - GLSL файлы вершинного и фрагментного шейдеров.

Используемые технологии:
OpenGL 3.3 Core Profile - рендеринг 2D/3D объектов.
GLFW - создание окна и обработка ввода.
GLAD - загрузка OpenGL функций.
GLM - работа с векторами и матрицами.
C++17 - основной язык реализации.

Цель проекта:
Демонстрация навыков работы с OpenGL, VBO/VAO, матрицами трансформаций и рендерингом динамических объектов, а также создание мини-сцены с управляемыми частицами.

Пример использования:
При запуске программы на экране отображается ядро и множество частиц, которые спавнятся и движутся по сцене с учётом времени и направления. Цвет и положение объектов обновляются динамически через uniform-переменные шейдера.

GLAD в связке с GLFW используется для загрузки и инициализации
указателей на функции OpenGL во время выполнения. Поскольку
функции OpenGL различаются в зависимости от ОС и драйверов 
видеокарты, GLAD (GL Loader) находит адреса этих функций, 
позволяя использовать их в коде, а GLFW создает окно и контекст.
